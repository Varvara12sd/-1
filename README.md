# Лабораторна робота №7: Наповнення та керування базою даних

## 1. Вербальна модель предметної області
Предметна область охоплює процес нарахування заробітної плати на підприємстві з гнучкою системою надбавок. Система керує наступними аспектами:

*   **Співробітники:** Облік персональних даних працівників, їхніх посад та історії кар'єрного росту.
*   **Посади:** Перелік посад у штатному розкладі з визначеними базовими окладами.
*   **Надбавки:** Система додаткових виплат (фіксованих сума або відсотків).
*   **Бізнес-правила:** 
    *   Обмеження призначення надбавок лише для відповідних посад.
    *   Контроль суми всіх активних надбавок (не повинна перевищувати базовий оклад).
    *   Автоматичне скасування надбавок при звільненні або несумісності при переведенні на іншу посаду.

## 2. Логічна та фізична ER-модель
База даних складається з 5 взаємопов'язаних таблиць:
*   `Positions` (Посади)
*   `Employees` (Співробітники)
*   `Allowances` (Типи надбавок)
*   `PositionAllowances` (Допустимість надбавок для посад)
*   `EmployeeAllowances` (Історія призначення надбавок співробітникам)
## 3. Опис реалізації наповнення
Для генерації великого обсягу даних (1001 запис) використано процедурний підхід T-SQL:
1.  Створено табличні змінні `@FNames`, `@LNames`, `@SNames` та `@Streets` для імітації реальних даних.
2.  Використано функцію `ABS(CHECKSUM(NEWID()))` для випадкового розподілу посад та типів надбавок.
3.  Для кожного співробітника автоматично згенеровано унікальний номер телефону та корпоративну пошту на основі ітератора `@i`.
